# 8.4 三层扩展机制总结 — 延展极限的降维阶梯

> 从随身便签到集装箱货轮再到跨次元任意门——Claude Code 的三层扩展机制为大模型 AI 生态交互蓝图刻画出了手术刀般精确的演进法则。

## 三层总览

| 扩展层级 | 隐喻 | 核心架构价值 |
|---------|------|------------|
| **Layer 1: Skills** (纯文本闭包) | 随身的便签本 | **零阻尼战场抢滩** — 基于纯 Markdown 快速试错迭代，将日常复杂要求统编为无状态"一次性冲锋组" |
| **Layer 2: Plugins** (硬装甲沙箱) | 冷酷的集装箱货轮 | **铁腕级系统统治** — Git/NPM 双通版本锁死依赖，配合连坐卸载的黑名单惩戒网 |
| **Layer 3: MCP** (极客协议底板) | 跨次元的任意门 | **无限化降维打击** — 以 400KB+ 代码铸就通信枢纽，碾碎宿主语言与异构算力的隔离牢笼 |

## 各层对比

| 维度 | Skills | Plugins | MCP |
|------|--------|---------|-----|
| **物理形态** | 纯 Markdown 文件 | NPM/Git 包 + plugin.json | JSON 配置 + 远端服务 |
| **加载方式** | `readFile` + 正则提取 | `npm install` / `git clone` | 多协议握手（stdio/sse/ws） |
| **代码位置** | 本地文件系统 | 本地缓存 + 远端仓库 | 远端异构环境 |
| **安全防线** | 路径优先级 + 软链接检查 + Shell 逃逸隔离 | Marketplace 心跳 + 黑名单连坐 | OAuth 401 拦截 + 体积截断 + 静默重连 |
| **生命周期** | 无状态，即生即灭 | 版本锁定，缓存持久化 | Session 管理，断点续传 |
| **扩展自由度** | 极高（写个 .md 就行） | 中等（需遵守包规范） | 取决于远端服务能力 |
| **代码量** | ~500 行 | ~4,000 行 | ~40,000+ 行 |

## 设计哲学：同心圆式安全外溢

三层机制遵循一条隐形法则：

> **离核心越近，操作约束越少；离核心越远，通信标准越严。**

- **Skills** 距核心最近——纯文本注入 Prompt，几乎零约束，但也只能影响大模型的"想法"
- **Plugins** 居中——能挂载代码和 Agent，但被包管理器和生态雷达严格看管
- **MCP** 距核心最远——能连通全球任意异构服务，但必须穿越 400KB 的协议网关、TLS 加密、OAuth 认证、体积海关

这种设计使得系统在保全核心稳定性的同时，可以无限向外触达未知的技术群岛。
